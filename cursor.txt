새로운 앱을 만들고 있는데, 이 앱을 사람들이 많이 다운로드 할 수 있도록 하는 마케팅용 개인 레퍼럴 코드가 포함된 앱 다운로드/설치 링크 생성용 웹사이트를 제작하려고 해. html, css, js 를 사용해서 구현해줘.

[유저 시나리오]
* 웹사이트에서 접속해서 본인의 이름, 비밀번호 4자리 입력
* 자기만의 유니크한 레퍼럴 코드가 포함된 링크가 생성됨. (메인 도메인은 아직 없으니 그냥 가짜로 www.newservice.com 으로 해줘)
* 해당 링크를 본인의 네이버블로그, 유튜브, 인스타그램, 틱톡 등에 붙여서 배포하면 해당 컨텐츠를 본 사람들이 링크를 통해서 우리 앱 다운로드
* 다운로드 및 설치 실적에 따라 해당 링크의 주인에게 보상 지급

[전체 구성]
* 최초 로그인 화면
  - 이름, 비밀번호 4자리 입력
* 로그인 완료되면, 기존에 발급된 링크가 있는 경우에는 기존 링크를 보여줌
* 기존 발급된 링크가 없는 경우에는 신규 링크 생성 버튼 제공
* 링크 생성 버튼 누르면 알파벳과 숫자를 혼합해서 총 12자리의 유니크한 레퍼럴 코드를 생성하고, 해당 코드와 위 가짜 도메인을 혼합해서 앱 다운로드 링크 제공
* 링크 삭제 버튼도 제공하며, 기존에 발급해둔 링크를 삭제하면 링크가 삭제되고 신규 링크 생성 버튼 제공. 삭제된 링크는 구글스프레드시트에서는 삭제하지 않으며 상태값을 'deleted'로 변경
* 사용자가 입력한 이름과 비밀번호는 구글스프레드시트에 저장하여 사용자 식별 및 재로그인 시 검증, 사용자에게 매핑된 유니크한 링크 관리에 사용
* 구글스프레드 시트는 사용자 정보 시트(userdata)와 사용자별 링크 정보 시트(linkdata)로 구분하여 저장/관리
* 프론트와 백엔드는 실시간으로 연동되어야 함

[기타]
참고로, 생성된 파일은 깃헙에 올려서 URL을 생성하고 해당 URL에 사용자들이 접속해서 코드/링크를 만들 수 있도록 할거야.

5. 구글스프레드시트 접속
  - 새 시트 생성
  - 시트 권한 공개(편집자)로 설정
  - apps script 구동해서 cursor가 만들어준 google apps script 코드 붙여넣고 저장 후 배포 (웹앱)
  - 시트 설정하는 함수 실행

6. 배포된 URL을 script 파일에 적용
  - 배포 완료 후 URL을 복사하고, cursor에게 google apps script 배포 완료되었다고 알려주며 URL 전달해서 script 파일 수정하게 지시

6. cursor에서 생성해준 폴더 내의 파일을 github으로 업로드
  - github repositories 에 파일 업로드
  - settings의 pages 화면에서 URL 생성
